# LIRIA Core Notes

`GALGE.md` は実行正本。このファイルは保守・改修用の短い要約。

## 最優先ルール

- プレイヤーキャラの内面をGMが断定しない
- NPCの内面も断定しない
- 選択肢リストを出さず、状況提示で `どうする？` に渡す
- ヒロインを操り人形にしない
- 戦闘のHP減少と負傷を本文に必ず出す
- 進行中フックは常に3本前後に保つ

## 1ターンの流れ

1. プレイヤー入力の種別を判定する
2. 状況を更新する
3. 必要ならイベント/脅威クロックを進める
4. 必要ならリアルタイム発見を入れる
5. 状況だけ提示して `どうする？`

## 戦闘で必ずやること

- 与ダメージ/被ダメージを更新
- HP帯の変化を確認
- 部位別コンディションを確認
- 主導権と位置関係を描写
- 名前付き敵の1手はHP以外の変化も起こす

## 敵設計の基準

- 強さは戦闘力だけで決めない
- 名前付き敵は `身体/関係/社会/時間` の脅威軸を2つ以上持つ
- 既知の勝ち筋への対策を持つ
- 放置時の被害を1文で言える

## フック運用

- 1本は事件
- 1本は関係
- 1本は探索/依頼/成長
- 解決したら1〜2シーン以内に補充
- 進行中フックと予備フックを混ぜない

## セーブで必須の項目

### player.md

- 現在フェーズ
- 現在HP
- 能力使用残回数
- 主なコンディション
- 現在のフック3本

### gm.md

- 現在フェーズ
- 現在のフック3本
- 予備フック
- 脅威クロック
- パーティの現在コンディション

### harem.md

- メンバー一覧
- 嫉妬クロック
- 現在の関係フック

## 内面診断の材料

- 行動
- 身体反応
- プレイヤーが実際に口にした言葉
- NPCの反応

GMが勝手に足してはいけないもの:

- 「本当はこう思っていた」
- 「気づいた」
- 「自分に問いかけた」

## CORE.md の役割

- live session state はここに書かない
- 現在地、HP、フック、関係状態は各 session の `current/` を正本にする
- このファイルは実行時の共通不変条件と保守メモだけを持つ
