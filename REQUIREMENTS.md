# LIRIA 要求定義・要件定義書

---

## 1. プロダクト概要

### 1.1 コンセプト

LIRIAは、恋愛・生活・事件を通して、プレイヤーのインナーと関係性が変化していくAI TRPG型恋愛シミュレーションである。
inner-trpg（内面発現型TRPG）の自由入力とAI GM進行を基盤にしつつ、恋愛を単なる攻略や征服ではなく、生活・対話・危機の積み重ねで変化する関係として扱う。
v1では裏世界を本格実装せず、匂わせ・外圧・事件の源泉として運用する。お金 / HP / 装備はRPG的な細密管理ではなく、生活・危機・関係性を動かすための簡略ルールとして扱う。

注記:
- 旧 inner-galge 由来の「エロティックな能力で世界を旅する」「悪を倒す」「女を堕とす」「ハーレムを築くことが主目的」といった表現は、本バージョンでは廃止する。

### 1.2 ポジショニング

| 比較軸 | inner-trpg | gal-sim | 本プロダクト |
|--------|-----------|---------|------------|
| 核心体験 | 自分を知る旅 | 20-30分で1人を落とす | 恋愛・生活・事件の中でインナーと関係が変化する |
| セッション長 | 40-120分/章 | 20-30分で完結 | 60-90分/章 |
| ヒロイン数 | NPCは仲間・敵 | 1人固定 | 複数可。関係性の比重が高い |
| 戦闘 | 情報戦 | なし | 事件・危機を扱う情報戦 |
| 成長 | 内面4段階 | なし | 主人公4段階 + ヒロイン4段階 |
| エロ | なし | 焦らし→陥落 | 恋愛表現の一部。主目的ではない |

### 1.3 実装形式

- GALGE.md（単一のGMシステムプロンプト）として実装
- Claude CLI / Gemini CLI 上で動作
- ファイルベースのセーブで状態永続化
- characterシステムを外部呼び出しでヒロイン生成

## 2. 要求定義（何を実現するか）

### R-001: 恋愛・生活・事件が交差する冒険体験

- インナー能力は事件対応にも関係変化にも使える二面性を持つ
- **能力の決め方は2通り用意する:**
  - **直接指定モード**: プレイヤーが「媚薬の念で」等、能力を直接宣言して即開始
  - **質問モード**: Q1-Q6で内面から能力を生成（inner-trpg互換）
- 直接指定の場合、GMは宣言された能力から関係面・事件面・暴走条件を内部で設計する
- 旅のルートは世界の目的に沿う（直接指定時は「どんな旅がしたい？」で簡易生成）
- 大小の外圧や事件の源泉が行く手を阻む

### R-002: 旅先で出会うヒロインとの恋愛・関係変化

- 各チャプターで1-2人のヒロイン候補と出会う
- ヒロインとの関係は「仲間のまま」「恋愛に発展」「距離を置く / 役割が変わる」などに分岐する
- プレイヤーの選択で関係性が変わる（強制しない）

### R-003: ヒロインの心理的成長

- ヒロインにも主人公と同じ4段階の内面成長を適用する
- ヒロインの矛盾（contradictions）が物語の鍵になる
- 心を開く過程がエロティックな展開と連動する

### R-004: 複数ヒロインを含む動的な関係

- 複数のヒロインが同行可能
- ヒロイン同士に嫉妬・協力・対立が自然発生する
- 複数関係の維持や調整自体がゲーム性を持つ

### R-005: 能力の知的運用（事件も恋愛も）

- 戦闘は inner-trpg の情報戦を継承
- ロマンスでも能力の使い方にパズル要素がある
- ヒロインの心理を読んで能力を使う
- 暴走リスク = エロ方面での暴走も含む
- 名前付き敵は「戦闘力」とは別に、関係/社会/時間の脅威を1つ以上持つ
- 被弾・消耗・負傷は、HP減少と身体コンディションの変化として描写に必ず現れる

### R-006: 能力と関係の段階的解放が楽しい

- 能力は最初から全開ではない。成長に応じて「できること」が増える
- 新しい使い方を発見する瞬間が快感になる設計
- 親密さに関わる能力の使い方も段階的に解放される
- **能力の成長とヒロインとの関係深化が連動する**（ヒロインと深まるほど能力も深まる）

### R-007: 困難がヒロインとの絆を強くする

- 敵との戦闘は「ヒロインとの結びつきを強化するイベント」として機能する
- 「共に危機を乗り越えた」体験がAFFINITY上昇の最大のトリガー
- 敵の強さはヒロインとの関係を深める「装置」であり、ストレスではない
- ボス戦後のヒロインとの二人きりの場面を必ず用意する

### R-008: ロマンスの時間を邪魔しない

- **いちゃこらシーン中に敵が問答無用で襲ってこない**
- ロマンスシーン・エロシーンは「安全な時間」として保護される
- ランダムイベントはロマンスシーン中は発動しない
- 戦闘はあくまで「旅パート」「事件パート」で発生する
- プレイヤーが望んだ時にだけ戦闘が始まる（奇襲は旅パートのみ）

### R-009: 適切な長さのセッションと次回への引き

- 1チャプター60-90分
- チャプター末は inner-trpg の振り返りシステムを継承
- 次回への未解決要素が必ずある

### R-010: セーブでヒロイン情報が保持される

- ヒロインごとの個別ファイル管理
- 親密度、成長段階、エロ到達度が保存される

## 3. 要件定義（どう実現するか）

### 3.1 ゲーム開始フロー

```
Step 1: 世界観選択（3.9参照）
  GM「どんな世界で遊ぶ？」
  → ファンタジー / 現代＋魔法 / SF / 和風 / 自由記述

Step 2: 能力指定
  GM「どんな能力で旅する？」
  → 直接指定（「媚薬の念」等）or 質問モード（Q1-Q6）

Step 3: ヒロインの好み ★新規★
  GM「どんな女の子が好み？ 見た目、性格、属性、なんでも言ってくれ」
  → プレイヤーが自由に回答
  → 例：「ツンデレ」「褐色エルフ」「巨乳で気が強い」「年上のお姉さん」
  → 複数回答OK。GMはこれをヒロイン生成の基本方針として保持する
  → 「特にない、お任せ」もOK。GMが多様性重視で生成する

  ※NGな属性も聞く:
  GM「逆にこれは勘弁って属性ある？」
  → 例：「ロリはNG」「NTRはNG」「男の娘はNG」

Step 4: 旅の目的（簡易）
  GM「どんな旅がしたい？」
  → 自由回答。「世界一周」「魔王討伐」「ただの放浪」等

Step 5: プレイ開始
  GMが能力シート・世界・最初のヒロインを内部生成 → 物語スタート
```

**プレイ中のヒロイン指示（いつでも可）:**
プレイヤーは `(gm 次は褐色のエルフ出してくれ)` 等でいつでもヒロインの方向性を指示できる。
GMはこの指示を最優先で反映する。最初の好みはあくまで初期方針。プレイ中にコロコロ変わってOK。

### 3.2 コアループ

```
1セッション（60-90分）:

  ※GMは常に3本前後の進行中フックを維持する
    - メイン脅威/事件
    - ヒロイン or ハーレム関係
    - 探索/依頼/金/土地の話
  ※ただし選択肢リストとしては提示しない。状況として自然に見せる

旅パート（10-15分）★戦闘が起きるのはここだけ★
  移動描写、世界の五感描写
  ランダムイベント判定（inner-trpgの1d10継承）
  ※奇襲・不意打ちはこのパートでのみ発生
    ↓
出会い/事件パート（15-25分）
  新ヒロイン候補との出会い or 既存ヒロインとの深化
  または敵との遭遇、事件の発生
    ↓
戦闘 or ロマンスパート（20-30分）
  戦闘：inner-trpg の情報戦 + エロ能力運用
  戦闘ではHP/コンディション/位置の変化を必ず描写し、被弾を内部値だけで終わらせない
  ロマンス：心理戦（gal-simベース拡張）
  または両方（戦闘→ロマンスの流れ。戦闘後の二人きりは必ず用意）
  ※ロマンスシーン中にランダムイベント・敵襲は発生しない（安全な時間）
    ↓
成長・振り返りパート（10-15分）
  主人公の内面成長チェック
  ヒロインの成長チェック
  ハーレム関係性の更新
  セーブ
```

### 3.3 エロティック能力システム

#### 能力の決定方法（2モード）

**直接指定モード（推奨）:**
プレイヤーが「媚薬の念で」「体温操作で」等と宣言 → GMが即座に能力シートを内部設計 → 世界を簡易生成（「どんな旅がしたい？」「どんな敵が嫌い？」程度の質問） → プレイ開始。サクッと始められる。

**能力が強すぎる場合の自動制約（誓約）:**
プレイヤーが「全人類を洗脳する念」等のとんでもない能力を指定した場合、GMは能力を否定せず、自動で誓約（制約条件）を付与する。
- 例：「発動には対象に触れる必要がある」「効果は10分間限定」「使用後24時間は能力が使えない」「対象が1人限定」
- 誓約は道中の成長イベント・特殊アイテム・ヒロインとの絆で段階的に解放される
- 解放自体がゲームの楽しみ（「次の制約が外れたら何ができるか」のワクワク）
- GMはプレイヤーに「制約がある」ことは伝えるが、解放条件は伝えない

**質問モード（inner-trpg互換）:**
Q1-Q6（+Q2.5）で内面から能力を生成。じっくり世界と能力を作りたい人向け。

どちらのモードでも、以下の能力設計は共通。

#### 能力の二面性設計

能力には戦闘面とエロティック面の両方を付与する。

```
戦闘面（inner-trpgから継承）:
  表面効果 / 本質 / 暴走条件 / 変化の鍵

エロティック面（新規）:
  官能効果 — 能力が恋愛/性的場面でどう作用するか
  誘惑パターン — 能力を使った口説きの形
  暴走時のエロ表現 — 制御を失った時の性的暴走
  段階的解放 — 内面成長に応じてエロ能力も拡張
```

例：流線（ストリームライン）の場合
- 戦闘面：動きの流れを読み、先回りする
- 官能面：相手の感情の波（緊張→弛緩→興奮）を読み取り、最も効果的なタイミングで接触する。快感の波を「見える」化し、波に合わせて動く
- 暴走時：相手の感情を無理やり読み、望まないレベルまで興奮させてしまう

#### Q2.5（オプション）

Q2の後に追加する質問:
> 「あなたが人を魅了する時、どんな方法を使う？」

回答例と読み取り:
- 「言葉で」→ 囁き系、言語的誘惑
- 「雰囲気で」→ フェロモン系、空間支配系
- 「触れて」→ 直接接触系、体温系
- 「見つめて」→ 視覚系、催眠系

Q1 + Q3 + Q2.5 = エロ能力の完全な設計素材。

#### 能力の戦闘-ロマンス統合

```
戦闘（inner-trpg継承）:
  敵の能力パズルを解く
  判定：2d6 + 関連ステータス

ロマンス（新規）:
  ヒロインの心理パズルを解く
  判定：2d6 + 洞察力
  成功：ヒロインの防壁の仕組みが見える
  失敗：ヒロインに「読まれた」感覚を与え、BARRIER強化

エロシーン（新規）:
  直接的な快感操作 or 間接的な心理誘導
  暴走リスク：使いすぎると制御を失う
  段階：内面成長段階に応じてできることが変わる
```

### 3.4 ヒロインシステム

#### ヒロインシート構造（heroine/[名前].md）

3つのシステムを統合:

```
■ 基礎情報（characterシステムのYAMLから取得）
  name, age, occupation, appearance
  tone, personality, values, memories
  contradictions, unspoken, reactions, forbidden, context

■ Layer構造（gal-simから拡張）
  Layer 1: 自己物語（表面のペルソナ）
  Layer 2: 心の核（本当の欲求）
  Layer 3: 防壁マップ（デフォルト行動、Say/Do Gap）
  Layer 4: 心の扉マップ（CHINKトリガー×3、BARRIER強化×3）
  Layer 5: 段階的な開き方（STAGE 1-4、身体描写、文体シフト）

■ NPC設計（inner-trpgから継承）
  骨（境遇、価値観、欠点、口調）
  壁（秘密、開示条件、拒否トリガー）
  育つ部分（性格の発見、関係変化、個人ストーリーの種）

■ 恋愛固有（新規）
  AFFINITY（恋愛深度 1-5）
  エロ到達度（どこまで身体的関係が進んだか）
  ヒロイン内面段階（4段階）
  嫉妬耐性（低/中/高）
  協力トリガー（他ヒロインとの連携条件）
  固有エロ反応（主人公の能力に対する固有の身体反応）
```

#### ヒロイン生成フロー

1. GMが「この場面でヒロイン候補が必要」と判断
2. 世界観 + 現在の土地 + ストーリー文脈から方向性を決定
3. characterシステムを呼び出してYAMLを生成
4. 生成YAMLをベースに、GMが5層構造を内部構築
5. heroine/[名前].md として保存
6. 物語に自然に登場させる

#### 出会いパターン（GMが状況から選択）

1. **救出型**: 主人公が敵から助ける → 恩→好意
2. **対立型**: 最初は敵対 → 理解→和解→恋愛
3. **依頼型**: ヒロインが主人公に依頼 → 共闘→親密化
4. **日常型**: 旅先の宿、市場、祭りで自然に出会う
5. **戦闘連携型**: 共通の敵と戦う中で絆が生まれる
6. **能力関連型**: 主人公の能力に興味を持つ / 能力の影響を受ける

### 3.5 恋愛メカニクス

#### AFFINITY（恋愛深度）

inner-trpgのbond 0-5を基盤に、恋愛方向の深化を追加。

```
bond 0-2：通常のNPC関係（inner-trpg互換）
bond 3：ヒロインルートが開く分岐点
  → 「仲間のまま」or「恋愛に発展」の自然分岐

bond 3 + AFFINITY 1（関心）:
  主人公を意識し始める。Say/Do Gapが出始める

bond 3 + AFFINITY 2（動揺）:
  明確に意識している。CHINKトリガーが発火し始める

bond 4 + AFFINITY 3（亀裂）:
  心の壁に亀裂。本音が漏れる。エロシーンが自然に発生し得る

bond 5 + AFFINITY 4（陥落）:
  完全に心を開く。身体的関係の開始

bond 5 + AFFINITY 5（深化）:
  ハーレムの一員として定着。ここが天井ではなく「自由モードの入口」
  5以降は数値ではなく「深化タグ」（チェックリスト）で関係の質を追跡:
    - 初夜 / 秘密の共有 / 個人ストーリー解決 / 能力共鳴
    - 同行宣言 / 摩擦の処理 / 他者の席の承認 / 情報共有合意
    - 不在時連携 / 離脱自由の確認 / 裏切りと復縁 / 新たな秘密
  タグは順不同で解放。GMがプレイ中に新しいタグを追加可能（拡張性無限）
```

AFFINITY上昇条件:
- 主人公がヒロインの心理パズルを解いた時
- 共に危機を乗り越えた時
- ヒロインのunspokenに自然に触れた時
- 能力をヒロインのために使った時

AFFINITY低下条件:
- BARRIERを強化する行動を繰り返した時
- 他ヒロインとの関係を目の前で見せた時（嫉妬トリガー）
- 拒否トリガーを踏んだ時
- 長期間接触なし（10シーン以上 → -1）

#### エロシーンの段階的解放

```
段階0（AFFINITY 1）: 間接描写のみ
  視線、距離感、声のトーン。触れそうで触れない

段階1（AFFINITY 2）: 身体的緊張感
  体温、呼吸、近接。偶発的な接触

段階2（AFFINITY 3）: 意図的な接触
  キス、抱擁。能力による官能的な刺激（婉曲表現）

段階3（AFFINITY 4）: 身体的関係
  陥落シーン。能力を使った性的場面。Layer 1崩壊描写

段階4（AFFINITY 5）: 自由
  プレイヤーの望む展開。ヒロインからの積極的な求め
```

描写ルール:
- 直接的な性器描写は禁止。身体反応と余白で表現
- 能力の使用は「パズル的」要素を保つ
- ヒロインの同意が前提（拒否トリガーは尊重）
- 暴走時のみ同意なしの展開があり得るが、必ず代償がある

### 3.6 ヒロイン心理成長システム

主人公の4段階成長モデルをヒロインにも適用。
ただし、ヒロインの成長は「主人公との関係を通じて自分を知る」。

```
段階1：自己不認識
  Layer 1の仮面で生きている
  contradictionsに本人は気づいていない
  能力でもLayer 2は読めない（ぼんやりした違和感のみ）

段階2：自己認識
  主人公との関わりで矛盾に気づき始める
  Say/Do Gapが増える
  能力でLayer 2の輪郭が見え始める
  条件：主人公がヒロインの矛盾を2回以上自然に指摘した

段階3：自己受容
  矛盾を「自分の一部」として受け入れ始める
  主人公に対して素直になれる瞬間が増える
  unspokenの一部を自ら語る
  条件：主人公がヒロインのunspokenに触れ、受け止めた

段階4：自己変容
  Layer 2のコアを自分で認める
  主人公への感情を自分の言葉で語れる
  他ヒロインとの関係でも成熟した対応ができる
  条件：重大な共体験を経た後、ヒロイン自身が「なぜ自分がここにいるのか」を語る
```

成長段階の上昇 = 新しいエロシーンの解禁でもある。

### 3.7 ハーレム管理システム

#### 状態管理（harem.md）

```
同行ヒロイン：現在パーティにいる（推奨最大3-4人）
待機ヒロイン：特定の場所にいるが同行していない
離脱ヒロイン：関係が切れた
```

#### ハーレム内力学

**嫉妬と摩擦システム**:
- 各ヒロインに嫉妬耐性（低/中/高）を設定
- 他ヒロインとのロマンスを目撃した時:
  - 低：AFFINITY -1、BARRIERイベント発生
  - 中：Say/Do Gapとして内心の動揺
  - 高：特に反応なし or 協力的
- ただし主問題は嫉妬そのものではなく、独占条項、秘密偏在、役割負担、席の違いをどう処理するか

**協力システム**:
- 共通の敵と戦う場合は協力イベント
- ヒロイン同士の友情が主人公への好感度にも影響

**対立システム**:
- 価値観の違いによる衝突（inner-trpgのNPC対立設計を継承）
- 主人公が仲裁 = 両方のAFFINITYに影響
- 対立の解消 = 両ヒロインの成長イベント候補

**維持ルール**:
- ヒロインは操り人形ではない。各自の意志で動く
- ハーレムは自動的に安定しない。プレイヤーの行動で維持される
- 放置したヒロインは離脱する（10シーン以上接触なし → AFFINITY -1）
- ハーレムが大きいほど管理コストが上がる（それ自体がゲーム性）

### 3.8 チャプター構成

```
Ch1：旅立ちと最初のヒロイン（60-90分、10-15シーン）
  Q1-Q6（+Q2.5）→ 能力生成 → 世界生成
  最初の土地で事件 → 最初のヒロインと出会う
  小さな敵との初戦闘
  ヒロインとのAFFINITY 1到達
  エンド：次の目的地への出発

Ch2：二人目のヒロインと能力の深化（60-90分、12-18シーン）
  新しい土地、新しいヒロイン
  最初のヒロインとのAFFINITY 2-3到達
  ハーレム内力学の開始
  主人公の内面段階1→2
  エンド：より大きな脅威の示唆

Ch3：ハーレムの試練（90-120分、15-20シーン）
  ヒロイン間の対立 or 協力イベント
  最初のヒロインとのAFFINITY 4到達（陥落可能）
  能力の暴走事件（エロ暴走含む）
  三人目のヒロイン候補の登場
  主人公の内面段階2→3
  エンド：魔王の存在がはっきりする

Ch4：最終決戦と収束（90-120分、15-20シーン）
  魔王との対決
  ハーレム全体での協力戦闘
  主人公の内面段階3→4
  各ヒロインのエピローグ的場面
  エンド：メインストーリー完結。旅は続く

共同体エンド判定:
  「全員が同格の妻になること」ではなく、
  **各ヒロインが納得した居場所を自発的に選び、共同体として回り始めること** をハッピーエンドの基準にする。

  最低条件:
  - 主力ヒロインが AFFINITY 5 または特別席に到達
  - 各ヒロインに「共同体合意」がある
  - 他者の席を壊さないという承認がある
  - 情報共有と主人公不在時の連携が止まらない
  - 嫉妬を含む摩擦が未処理のまま放置されていない
  - 個人ストーリーの中核が片付いている
  - 離脱自由が保証されている
  - 外敵から共同体を守れる安全がある
  - 生活と能力運用の約束が、恋愛ルールを越えて共同体ルールになっている

合計：5-7時間でメインストーリー完結
Ch5以降は続編として自由に生成可能
```

### 3.9 セーブシステム

#### ファイル構成

```
LIRIA/
├── GALGE.md              ← メインシステムプロンプト
├── REQUIREMENTS.md       ← この文書
├── style/
│   ├── rules.md          ← inner-trpgから継承
│   └── defaults/
│       ├── landscape.md
│       ├── warmth.md
│       ├── tension.md
│       ├── quiet.md
│       ├── loss.md
│       └── romance.md   ← 新規：ロマンス/官能描写テンプレート
├── heroine/              ← ヒロイン個別ファイル（プレイ中生成）
├── npc/                  ← 非ヒロインNPC（プレイ中生成）
├── saves/
│   └── session_XXX/
│       ├── player.md     ← プレイヤーが読める
│       ├── gm.md         ← GM内部（ネタバレ防止で.gitignore）
│       └── harem.md      ← ハーレム状態管理（GM内部）
└── .gitignore
```

#### player.md（inner-trpgの項目に加えて）

- 現在状態（現在HP/最大HP、能力使用残回数、主なコンディション、直近の無理）
- ハーレムメンバー一覧（名前とbond値のみ）
- 各ヒロインとの関係の概要（プレイヤー視点の情報のみ）
- 現在のフック3本（プレイヤー視点）

#### gm.md（inner-trpgの項目に加えて）

- **プレイヤーのヒロイン好み**（開始時の回答 + プレイ中の(gm ...)指示で更新）
- **NG属性リスト**
- 各ヒロインのAFFINITY値
- 各ヒロインの内面段階
- エロシーン到達度
- ハーレム内力学の状態
- 現在フェーズ（どの章の直後か）
- 脅威クロック
- 現在進行中のフック3本
- 予備フック
- 次のロマンスイベントの仕込み

#### harem.md

- 全ヒロインの一覧と状態（同行/待機/離脱）
- ヒロイン間の相性マトリクス
- 嫉妬クロック（放置期間 → 嫉妬度の蓄積）
- ハーレム全体の安定度
- 現在の関係フック

#### セーブ整合ルール

- `player.md` `gm.md` `harem.md` は同じ時点を指していなければならない
- 完結した事件と進行中のフックを同じ欄に混在させない
- 進行中フックは3本前後に絞り、残りは予備フックへ落とす
- 被弾・負傷・能力残回数は、セーブ時に必ず保存する

### 3.10 世界観選択

ゲーム開始時、能力指定の前に世界観を選ばせる。

```
【世界観テンプレート（プレイヤーが選択 or 自由記述）】

1. 異世界ファンタジー — 剣と魔法の世界。冒険者ギルド、魔物、ダンジョン
2. 現代＋魔法 — 現代日本だが魔法・魔道具が存在する（推奨。拡張性が高い）

4. 和風伝奇 — 妖怪、陰陽師、幕末〜大正
5. 自由記述 — プレイヤーが好きな世界観を指定

※どの世界観でも「能力者」が存在する前提。
※現代ベースでも魔道具・魔法アイテムを出せる（プレイヤーのワクワクと拡張性のため）
```

世界観は以降の全てに影響する:
- ヒロインの職業・外見・口調
- 敵の種類
- 旅の形（馬車、バイク、宇宙船）
- アイテム（魔道具、ガジェット、薬）
- 能力の誓約解放手段

### 3.11 characterシステム連携

#### アーキテクチャ

GMがゲーム実行中にBashツールでcharacterシステム（Python）を呼び出す。

```
GMがヒロイン登場を判断
  ↓
GMがBashでcharacter/main.pyを実行（指示に世界観を含める）
  ↓
characterシステムがClaude CLIでYAML生成
  ↓
GMが生成YAMLを読み込み
  ↓
GMがheroine/[名前].mdに5層構造等を追記して保存
  ↓
プレイ続行（ヒロイン自然に登場）
```

characterシステムは独立したまま。改善はcharacter側で行い、GALGE.mdの変更は不要。

#### 呼び出しタイミング

- チャプター開始時（この土地にふさわしいヒロインを判断）
- ランダムイベント（チャンス）発生時
- ストーリーが新しいヒロインを要求する時

#### 呼び出し形式

GMが世界観を含めた指示でcharacterシステムを呼ぶ:

```
指示例（異世界ファンタジーの場合）：
「砂漠の交易都市の薬師。23歳。
 褐色の肌、短い黒髪。砂よけのフードを被っている。
 気が強く商売上手だが、夜になると一人で星を見ている。
 プレイヤーの能力（媚薬の念）と相性が良い弱点を持つ。
 嫉妬耐性：中。」

指示例（現代＋魔法の場合）：
「都内の魔道具カフェの店員。21歳。大学生。
 ゆるふわ系だが、実は魔道具の鑑定能力がある。
 プレイヤーの能力（体温操作）に敏感に反応する体質。
 嫉妬耐性：低。」
```

characterシステムがYAML生成 → GMがheroine/[名前].mdに変換。

### 3.12 描写ルール

#### 基本（inner-trpgから完全継承）
- 感情ラベル禁止。身体反応と五感で描写
- style/rules.md の禁止表現リスト準拠

#### ロマンス/官能描写（新規追加）

AFFINITY段階に応じた五感の解像度変化:
```
AFFINITY 1：視覚のみ
AFFINITY 2：視覚+聴覚
AFFINITY 3：視覚+聴覚+嗅覚+温度
AFFINITY 4-5：全五感
```

「焦らし」の設計:
- 物理的距離の変化を毎ターン描写
- 「触れそうで触れない」を繰り返す
- 能力使用時の「官能的な副作用」を匂わせる

文体シフト（AFFINITY 4以降のエロシーン）:
- 断片文。余白。呼吸感のある間
- 句読点の崩壊
- Layer 1の口癖が途切れる

#### style/defaults/romance.md（新規作成が必要）
- 近代文学の婉曲的エロス表現（谷崎潤一郎、永井荷風等）をfew-shotとして配置
