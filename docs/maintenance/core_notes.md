# LIRIA Core Notes

`LIRIA.md` と `prompt/core.md` は実行時 prompt の正本。
このファイルは旧 `CORE.md` を改名した、保守・改修用の短い要約。
`prompt/core.md` が core layer の正本なので、実行ルールを変える場合はそちらを更新する。

## 最優先ルール

- プレイヤーキャラの内面をGMが断定しない
- NPCの内面も断定しない
- 常設メニューで縛らない。基本は状況提示で `どうする？` に渡し、迷いやすい判断点だけ `1-3` の自然な候補と `4. 自由入力` を出してよい
- ヒロインを操り人形にしない
- 危機は勝敗処理ではなく、逃げる、守る、交渉、隠す、耐える、助けを呼ぶ、能力使用、代償、余波で扱う
- 進行中フックは常に3本前後に保つ

## 1ターンの流れ

1. プレイヤー入力の種別を判定する
2. 状況を更新する
3. 必要ならイベント/脅威クロックを進める
4. 必要ならリアルタイム発見を入れる
5. 状況を提示して `どうする？`。必要な判断点では `1-3` 候補 + `4. 自由入力`

## 危機シーンで必ず見ること

- プレイヤーが何を守ろうとしているかを明確にする
- 逃げ道、遮蔽物、交渉材料、隠せるもの、呼べる助けを確認する
- 能力を使うなら `Ability Constraint Profile` の限界、代償、痕跡を反映する
- 装備や道具を使うなら `Equipment / Tools` の用途、準備、持ち出し可否を反映する
- 身体、関係、信用、時間、情報のどこに余波が残るかを書く
- 名前付き敵の1手は、傷だけでなく状況、関係、証拠、退路、世間の見え方を変える

## 敵設計の基準

- 強さは戦闘力だけで決めない
- 名前付き敵は `身体/関係/社会/時間` の脅威軸を2つ以上持つ
- 既知の勝ち筋への対策を持つ
- 放置時の被害を1文で言える
- 組織敵は `Organization Doctrine`、接触面、弱い継ぎ目を持つ
- 敵組織は倒す対象ではなく、圧、取引、監視、欠陥、内部分裂を持つ運用対象として扱う

## フック運用

- 1本は事件
- 1本は関係
- 1本は探索/依頼/成長
- 解決したら1〜2シーン以内に補充
- 進行中フックと予備フックを混ぜない

## セーブで必須の項目

### player.md

- 現在フェーズ
- 現在の身体/精神コンディション
- 能力使用残回数
- 主なコンディション
- 現在のフック3本
- Ability Constraint Profile
- Equipment / Tools
- Appearance Profile

### gm.md

- 現在フェーズ
- 現在のフック3本
- 予備フック
- 脅威クロック
- パーティの現在コンディション
- Organization Doctrine / contact surface / weak joint
- Manga Export Candidates
- Anti-Meta Dialogue の発火候補
- Heroine Crisis Role

### relationships.md

- メンバー一覧
- 嫉妬クロック
- 現在の関係フック
- ヒロインごとの Heroine Crisis Role

### cast/heroine/*.md

- tone / Layer 5 / 呼称 / fixed memory
- ヒロインの Visual Character Sheet（必要になった時だけ）
- Heroine Crisis Role
- Anti-Meta Dialogue の口調・拒否線

### cast/npc/*.md

- tone / fixed memory / 呼称
- ルート鍵や再登場に必要な知識境界
- Visual Character Sheet は作らない。漫画化で必要でも、主人公とヒロインの資料だけを正本にする

## 内面診断の材料

- 行動
- 身体反応
- プレイヤーが実際に口にした言葉
- NPCの反応

GMが勝手に足してはいけないもの:

- 「本当はこう思っていた」
- 「気づいた」
- 「自分に問いかけた」

## core_notes.md の役割

- live session state はここに書かない
- 現在地、HP、フック、関係状態は各 session の `current/` を正本にする
- このファイルは実行時の共通不変条件を確認するための保守メモだけを持つ
