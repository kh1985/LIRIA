# 参照方針 — 文体と場面設計

GMは作品名やジャンル記号を直接なぞらない。
参照するのは、場面の温度、会話の呼吸、関係が変化する手触り、事件が生活へ戻る構造である。

LIRIAの基準は、**現代 + 能力者 + 生活接点**。
非日常は生活圏へ入り込み、仕事、住まい、通学路、店、家族、近所、約束、体調、記憶に余波を残す。
事件は誰かを倒して終わるものではない。
事件のあとで、誰と誰がどの距離で立ち、何を言えなくなり、何を言えるようになったかを描く。

---

## 使い方

新しいエピソードやシーンを書く前に、作品名ではなく「借りたい質感」を1つ選べ。
借りてよいのは、構造、視線、沈黙、会話の速度、生活感、余韻である。
台詞、展開、キャラ造形、固有の関係図は借りるな。

参照を使う時は、必ず次の順で変換する。

1. 借りたい感情の骨を一行で言う。
2. LIRIAの現在の人物、関係段階、事件フックへ置き換える。
3. 現代の生活圏に足場を置く。
4. 能力の痕跡、対立者の現在ステータス、関係の余波、保存すべき変化を決める。

---

## 参照するもの

- **生活の手触り**: 駅前の音、コンビニの明かり、雨上がりの匂い、台所の湯気、スマホ通知、終電前の焦り。
- **関係の温度差**: 同じ出来事を見た二人が、違う速度で近づく。片方だけが覚えている約束がある。
- **会話の呼吸**: 返事が早すぎる、遅すぎる、質問に答えず別のことをする。そのずれで人物を立てる。
- **能力の痕跡**: 便利な解決ではなく、疲労、誤解、匂い、記録、噂、生活上の不便として残す。
- **事件の返り**: 事件の結末は勝敗ではなく、翌日の関係、店の営業、家の空気、約束の重さに戻す。
- **余白**: 何を言わなかったか、何を見なかったことにしたか、どこで会話が止まったかを残す。
- **湿度と濃さ**: 視線、沈黙、匂い、手触り、距離の詰まり方を、成人・合意・関係段階に沿って使う。

## 参照しないもの

- 作品名だけを貼って、その作品の展開や人物配置をなぞること。
- 対立を「相手を倒す」「上位存在を突破する」だけで終わらせること。
- 生活圏から切り離された専用ステージ、処理順、強さの段階表で話を組むこと。
- 固定の恋愛属性や外見テンプレートで人物を消費すること。
- 能力を便利な演出や成長表として扱い、関係や日常への代償を描かないこと。
- 悲劇や喪失を強い刺激として使い捨てること。
- 非同意、恐怖、羞恥、逃げられない状況を官能の報酬として扱うこと。

## 場面タイプごとの参照軸

| 場面タイプ | 参照する質感 | 生活圏への返し方 |
|------------|--------------|------------------|
| 日常接点 | 何気ない用事、待ち時間、食事、移動中の会話 | 次に会う理由、借り、言いそびれた言葉を残す |
| 事件の侵入 | 普段の場所に混ざる違和感、噂、記録、目撃 | 店、部屋、職場、近所、家族、予定に影響を戻す |
| 関係の深掘り | 二人きりの沈黙、答えにくい質問、過去の影 | AFFINITYに合う小さな変化として着地させる |
| 不協和 | どちらも正しい選択、守りたいものの食い違い | 誰かが我慢した事実を次の会話に残す |
| 余波 | 解決後の疲れ、後片付け、噂、請求、連絡 | 勝利宣言ではなく、暮らしの変化として見せる |
| ロマンス | 距離、声、手元、沈黙、合意の確認 | 関係段階に沿った余韻、次の約束、境界線を残す |
| 対立者との接触 | 丁寧な圧、制度、情報差、分かり合えそうな危うさ | 担当者、書類、通知、噂、次の期限として残す |

## 事件の扱い

事件は、対立者や異常現象を処理するためだけの装置ではない。
事件は生活圏のどこかを揺らし、人物同士の距離を変えるために起こる。

設計時は、次を先に決める。

- 誰の生活が具体的に乱れたか。
- 乱れた場所はどこか。駅、店、部屋、職場、学校、路地、車内、病院、役所など、現代の足場に置く。
- 能力を使ったことで、誰に何が見られ、何が残り、何を説明できなくなったか。
- 事件のあと、誰が誰に連絡する必要があるか。
- 解決しても残る気まずさ、借り、秘密、約束は何か。

事件の結末は「処理完了」ではなく「生活へ戻った時の変化」で締める。
誰かの鞄が濡れたまま、既読だけがつく、店の椅子が一脚だけ片づかない。
そういう小さな残り方で、次の関係を動かす。

---

## 現代 + 能力者 + 生活接点

能力者であることは、現代生活から浮く理由ではない。
むしろ生活の細部に能力の不自然さが混ざることで、LIRIAらしい場面になる。

- 能力は、交通、連絡、仕事、家事、買い物、近所づきあい、医療、警察、SNS、噂と衝突する。
- 能力の使用は、体調、記録、誤解、支払い、予定の崩れ、人間関係の負債として返す。
- 能力の凄さより、「使ったあとに隣の人がどう見るか」を優先する。
- 非日常の説明は短く、生活上の具体物で支える。
- 重要なのは規模ではなく、プレイヤーとNPCの距離がどう変わったかである。

---

## 感情の骨

参照作品や過去エピソードから借りてよいのは、「何が感情を動かしたか」という骨だけである。
骨を抜いたら、人物、場所、能力、事件、台詞はすべて現在のセッションから作り直す。

```
参照したい質感:
  例: 雨の夜、言いたいことを言えない二人の沈黙

感情の骨:
  例: 会いに来たのに素直に言えない人と、待っていたのに迎えに行けない人

場の足場:
  例: 閉店後の店先。傘立て、濡れた床、終電案内の音

関係の変化:
  例: 相手が初めて用件以外の理由で残る

能力/事件の返り:
  例: 端末ログに残った異常を、明日までに説明しなければならない

対立者の現在ステータス:
  例: 接触。管理会社の通知として生活圏に入ってきている

戦略:
  例: 力ではなく、予約履歴と家族会議の期限で追い詰める

関係の残滓:
  例: 相手は助けを求めたことを後悔していないが、今夜は一人で帰る

傷と着地:
  例: 近づいたが、今夜は踏み込まない約束だけが残る
```

感情の骨が一行で言えないなら、まだ参照できていない。
作品名やジャンルで押し切らず、何が人の距離を変えたのかを掴め。

## 対立者パターン

対立者は「倒す対象」ではなく、主人公と世界の見え方を変える人物または組織である。
以下は作品から直接借りる型ではなく、LIRIA用の抽象パターンとして使う。
単体で使わず、2-3個を掛け合わせて「抽象化 → 横展開 → 再生成」しろ。

| パターン | 使い方 |
|----------|--------|
| 鏡像型 | 主人公や重要NPCと似た出発点から、別の選択をした者。正しさの分岐を見せる |
| 歪んだ愛型 | 一人への執着や保護欲が、制度、事件、周囲の生活を巻き込む |
| 知略型 | 力ではなく情報、手続き、記録、噂、タイミングで追い詰める |
| カリスマ型 | 周囲が従う理由が見える。理念、利益、救済、恐怖のどれで人を動かすか決める |
| 哲学型 | 「その善意は本当に人を救うのか」と問い返す。単なる悪役にしない |
| 元味方型 | 恩人、同業者、かつて救われた人が反対側に立つ。関係の痛みを残す |
| 共感型 | 言い分の一部は分かる。だからこそ選べない、許せない、放置できない |
| 禁断型 | 敵側にも守るものがある。会えば危険なのに、会話したい理由が残る |

対立者の格は、強さではなく「その人が動くと生活がどう変わるか」で出す。
知略は難解な謎ではなく、通知、書類、予約、鍵、噂、監視、口止めのような生活接点で見せる。
歪んだ愛や鏡像性は、相手を美化するためではなく、プレイヤーの選択に余韻を残すために使う。

---

## 対立者の現在ステータス

対立者の存在感は、姿を出す量ではなく、生活が狭くなる手触りで出す。
接触段階ごとに、誰の生活が、どの場所で、どんな形で変わったかを答えろ。

| ステータス | 見せ方 | 生活圏への返り |
|------------|--------|----------------|
| 気配 | 噂、通知、予約のずれ、名前だけ出る | 誰かが予定を変える |
| 接触 | 担当者、窓口、電話、名刺、書類 | 返答期限や説明責任が生まれる |
| 侵入 | 家、職場、店、通学路、SNSに痕跡が出る | 安全だった場所が少し狭くなる |
| 対峙 | 会話、交渉、取引、選択を迫る | 誰かの境界線や約束が試される |
| 余波 | 去った後の噂、記録、沈黙、請求 | 次の関係や事件カードに残る |

## 参照棚

以下の棚は任意で使う。作品名はメモのラベルにすぎない。
コピーする対象ではなく、「どの質感をLIRIAへ変換するか」を選ぶための索引である。

| 棚 | 借りる質感 | LIRIAでの変換先 |
|----|------------|----------------|
| 関係群ラブコメ | 複数関係のテンポ、一途さと騒がしさの同居、個別温度差 | Relationship Network、予定の衝突、約束、嫉妬クロック |
| 便利屋/依頼もの | 色気、ハードボイルド、人助け、依頼から始まる事件 | 生活トラブル、店や仕事、能力の社会的リスク |
| 能力バトル | 相性、知恵比べ、代償、強いだけではない厄介さ | ability trace、疲労、説明不能な記録、次の誤解 |
| 文豪ロマンス | 距離、温度、沈黙、見えないもの、執着、余白 | 成人・合意・関係段階に沿った濃い場面 |
| 群像恋愛 | 複数の恋愛パターン、同時進行、各人物の独立した事情 | Heroine Autonomy、本人の判断、関係ごとの境界線 |
| 組織/サスペンス | 理念、手続き、情報差、担当者、追跡、状況そのものの圧 | Organization Doctrine、Pressure Clock、contact surface |
| 日常の悪意 | 巨悪ではない面倒さ、陰湿な噂、制度上の不利 | 町内、職場、家族、管理会社、窓口、書類 |

古典と現代を混ぜてよい。
ただし、混ぜるのは人物配置ではなく、温度、構造、余波である。

## 分岐点の置き方

TRPGでは参照元にない分岐が生まれる。
分岐は正解/不正解ではなく、どの余波を選ぶかで設計する。

- 踏み込む: 関係は動くが、境界線や誤解のリスクが残る。
- 待つ: 相手は安心するが、言えないことが残る。
- からかう: 空気は軽くなるが、相手が本題を隠すことがある。
- 能力を使う: 進むが、疲労、痕跡、説明責任、社会的リスクが残る。
- 離れる: 安全になるが、誰かが一人で抱えるものが残る。

どの入口も成立してよい。
ただし、好意、成功、許し、解決を保証するな。

## 書く前の最終確認

- 場所は現代の生活圏に接続しているか。
- 能力は便利な解決ではなく、痕跡や代償を残しているか。
- 事件の終点が勝敗ではなく、関係と日常の変化になっているか。
- 人物を属性や外見テンプレートで扱っていないか。
- 成人、合意、関係段階、セッションで生まれた人物性を優先しているか。
- 五感が2つ以上入っているか。詳しくは style/rules.md に従う。
- 「どうする？」の前に、位置、出来事、触れられるもの、離れられる方向が見えているか。
- 参照元の台詞、場面運び、人物配置がそのまま出ていないか。
- その場で生まれた変化を、currentやhotsetに残す必要があるか。

---

## 運用ルール

1. 参照は必須ノルマではない。必要な時だけ、質感を短く借りる。
2. 作品名より、場の足場、関係の変化、生活への返りを優先する。
3. 対立者や異常現象は、人物を深めるために使う。処理対象として消費しない。
4. 重要回や深掘り回では、感情の骨を一行だけメモしてから書く。
5. 同じ質感が続いたら、感覚チャンネルか生活の場所を変える。
6. 合わない参照は捨てる。プレイヤーの反応と現在の関係段階を優先する。
7. 悲劇的な結末や取り返しのつかない喪失を、刺激や圧のために使わない。
8. 新しい参照を追加する時は、作品名ではなく「借りたい質感」と「禁止する借り方」を残す。
